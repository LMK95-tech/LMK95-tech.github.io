01. Contexte & Problématique

La société SISR-Corp gérait son parc informatique et son support technique de façon entièrement manuelle et non structurée. L'inventaire du matériel (PC, serveurs, périphériques, licences logicielles) était maintenu dans des fichiers Excel partagés, mis à jour irrégulièrement et souvent obsolètes dès la semaine suivante. Les demandes d'assistance étaient reçues par téléphone ou e-mail sans aucun suivi structuré.

Périmètre SISR-Corp

  • Serveur : srv-glpi-01 (Debian 12)
  • IP : 192.168.29.82
  • Stack : Apache · MariaDB · PHP 8.2
  • Application : GLPI 10.x
  • Modules : Parc · Helpdesk · Agent

Problèmes identifiés

  • Inventaire Excel obsolète en permanence
  • Support e-mail sans historique ni suivi
  • Aucune statistique de pannes récurrentes
  • Shadow IT non contrôlé (USB, disques)
  • Fins de garantie découvertes trop tard
Scénario type d'incident sans GLPI
Utilisateur appelle le support Ticket perdu dans boîte mail Incident non résolu — utilisateur insatisfait

02. Solution retenue : GLPI 10

La solution déployée est GLPI 10 (Gestionnaire Libre de Parc Informatique), application open source ITSM de référence, hébergée sur un serveur dédié srv-glpi-01 sous Debian 12 avec la stack LAMP. L'agent GLPI installé sur chaque poste client remonte automatiquement les configurations matérielles et logicielles.
 Bénéfices de GLPI 10
Inventaire automatique — l'agent GLPI détecte CPU, RAM, disques, OS et logiciels installés sans saisie manuelle.
Ticketing ITSM — chaque demande est tracée avec priorité, catégorie, assignation et historique complet.
SLA configurables — délais de prise en charge et de résolution définis par catégorie avec relances automatiques.
Tableaux de bord — graphiques de tendances, temps moyens de résolution, équipements récurrents en panne.

05. Installation de la Stack LAMP

La première étape consiste à préparer le serveur srv-glpi-01 avec l'ensemble des prérequis logiciels. GLPI 10 requiert PHP 8.2 minimum avec un ensemble précis d'extensions. L'installation se fait via apt depuis les dépôts Debian 12 standard.
Bash / Debian 12 srv-glpi-01 — Installation Stack LAMP complète
# Mise à jour du système
root@srv-glpi-01:~# apt update && apt upgrade -y
# Installation Apache + MariaDB
root@srv-glpi-01:~# apt install -y apache2 mariadb-server
# Ajout du dépôt PHP 8.2 (Ondrej Sury)
root@srv-glpi-01:~# curl -sSL https://packages.sury.org/php/README.txt | bash -x
# Installation PHP 8.2 et toutes les extensions requises par GLPI 10
root@srv-glpi-01:~# apt install -y php8.2 php8.2-curl php8.2-gd php8.2-intl \
php8.2-mbstring php8.2-mysql php8.2-xml php8.2-zip \
php8.2-bz2 php8.2-ldap php8.2-apcu libapache2-mod-php8.2
# Activation des modules Apache nécessaires
root@srv-glpi-01:~# a2enmod rewrite
root@srv-glpi-01:~# systemctl enable --now apache2 mariadb
✓ Stack LAMP opérationnelle — Apache + MariaDB + PHP 8.2 actifs

06. Installation & Sécurisation GLPI 10

GLPI 10 est téléchargé depuis le dépôt officiel GitHub, extrait dans /var/www/html/glpi. La sécurisation post-installation impose le déplacement des répertoires sensibles config/ et files/ hors de la racine web — bonne pratique GLPI 10 contre les accès directs non autorisés.
Bash / Debian 12 Configuration MariaDB + Sécurisation GLPI 10
# Création de la base de données dédiée GLPI
root@srv-glpi-01:~# mysql -u root -p
MariaDB [(none)]> CREATE DATABASE glpi_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
MariaDB [(none)]> CREATE USER 'glpiuser'@'localhost' IDENTIFIED BY 'MotDePasse@2026';
MariaDB [(none)]> GRANT ALL PRIVILEGES ON glpi_db.* TO 'glpiuser'@'localhost';
MariaDB [(none)]> FLUSH PRIVILEGES; EXIT;
# Téléchargement et extraction GLPI 10
root@srv-glpi-01:~# wget https://github.com/glpi-project/glpi/releases/latest/download/glpi-10.x.x.tgz
root@srv-glpi-01:~# tar xzf glpi-10.x.x.tgz -C /var/www/html/
root@srv-glpi-01:~# chown -R www-data:www-data /var/www/html/glpi
# ── Sécurisation post-installation (GLPI 10 best practice) ──
root@srv-glpi-01:~# mv /var/www/html/glpi/config /etc/glpi
root@srv-glpi-01:~# mv /var/www/html/glpi/files /var/lib/glpi
root@srv-glpi-01:~# rm -f /var/www/html/glpi/install/install.php
# install.php supprimé → surface d'attaque réduite
✓ GLPI 10 installé et sécurisé — http://192.168.29.82/glpi/ opérationnel
 Sécurisation post-installation — points clés
install.php supprimé — empêche toute réinstallation non autorisée
config/ déplacé vers /etc/glpi — fichiers de config hors de la racine web
files/ déplacé vers /var/lib/glpi — uploads et cache hors de la racine web
Politique MDP admin — longueur min. 10, chiffres, maj, min, symboles obligatoires
