01. Contexte MCO & Problématique

L'ESN Aura-Technologie (80 collaborateurs) administrait son SI en mode totalement réactif : aucune métrique de performance (CPU, RAM, disque, trafic réseau) n'était collectée en temps réel. L'état de santé des équipements critiques — Active Directory, serveur de fichiers, routeur pfSense — était inconnu jusqu'à ce que les utilisateurs signalent un dysfonctionnement.

Infrastructure Aura-Technologie

  • Zabbix : Aurasvrzabbix (Debian 12)
  • IP serveur : 192.168.23.53
  • pfSense : 192.168.23.254 (SNMP)
  • Aura-AD : Windows Server (Agent)
  • Zabbix 7.0.25 LTS

5 problèmes opérationnels

  • Détection tardive des pannes (>2h)
  • Aucune métrique CPU/RAM/disque
  • Zéro alerting automatique
  • MCO non garanti — SLA impossibles
  • Post-mortem aveugle (pas d'historique)
Scénario type sans supervision
Disque AD se sature Découverte 3h après par un user Service AD coupé — Production arrêtée

02. Solution retenue : Zabbix 7.0 LTS

La solution déployée est Zabbix 7.0 LTS (Long Term Support), plateforme de supervision open source de référence en environnement professionnel. Zabbix 7.0 supporte nativement les protocoles SNMP v1/v2c/v3, les agents actifs et passifs, les checks ICMP et HTTP, et offre un moteur de corrélation d'événements avancé pour l'alerting.

05. Installation Zabbix 7.0 sur Debian 12

L'installation commence par l'ajout du dépôt officiel Zabbix 7.0 pour Debian 12 (Bookworm). Le paquet zabbix-release_7.0-1+debian12_all.deb configure les sources APT pointant vers repo.zabbix.com/zabbix/7.0/debian. Les composants serveur, frontend et agent sont ensuite installés avec MariaDB comme moteur de base de données.
Bash / Debian 12 Aurasvrzabbix — Installation complète Zabbix 7.0
# Ajout du dépôt officiel Zabbix 7.0
root@Aurasvrzabbix:~# wget https://repo.zabbix.com/zabbix/7.0/debian/pool/main/z/zabbix-release/zabbix-release_7.0-1+debian12_all.deb
root@Aurasvrzabbix:~# dpkg -i zabbix-release_7.0-1+debian12_all.deb
root@Aurasvrzabbix:~# apt update
# Installation du serveur, frontend Apache et agent
root@Aurasvrzabbix:~# apt install -y zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf \
zabbix-sql-scripts zabbix-agent2
# Création de la base MariaDB dédiée Zabbix
root@Aurasvrzabbix:~# mysql -u root -p -e "CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;"
root@Aurasvrzabbix:~# mysql -u root -p -e "CREATE USER 'zabbix'@'localhost' IDENTIFIED BY 'ZabbixAura@2026';"
root@Aurasvrzabbix:~# mysql -u root -p -e "GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';"
# Import du schéma initial Zabbix
root@Aurasvrzabbix:~# zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql -u zabbix -p zabbix
# Démarrage et activation des services
root@Aurasvrzabbix:~# systemctl enable --now zabbix-server zabbix-agent2 apache2

06. Configuration SNMP v2c — pfSense

Sur l'interface pfSense (192.168.23.254), le démon SNMP est activé dans Services → SNMP sur le port 161 (UDP). La communauté de lecture AuraPublic est définie — elle sert de mot de passe partagé entre Zabbix et pfSense pour les requêtes SNMP GET.
pfSense Services / SNMP — Configuration communauté AuraPublic
# Configuration pfSense — Services > SNMP
SNMP Daemon Enable : ☑ Enable the SNMP Daemon and its controls
Polling Port : 161
Read Community String: AuraPublic
SNMP Trap : Désactivé
# Macro Zabbix — {$SNMP_COMMUNITY} = AuraPublic
# Doit correspondre exactement à la communauté pfSense
# Test SNMP depuis Zabbix server
root@Aurasvrzabbix:~# snmpget -v2c -c AuraPublic 192.168.23.254 1.3.6.1.2.1.1.1.0

07. Triggers & Alerting — Seuils configurés

Les triggers (déclencheurs) Zabbix définissent les seuils d'alerte pour chaque métrique surveillée. Lorsqu'un seuil est franchi, Zabbix déclenche automatiquement une action — notification e-mail, création de ticket GLPI, ou exécution de script de remédiation.
Nom du déclencheurHôteExpression ZabbixSévéritéAction
Service AD DS arrêté Aura-AD last(/AuraAD/service.info[NTDS,state])<>0 Désastre Alerte immédiate
CPU > 90% pendant 5min Zabbix server avg(/Zabbix server/system.cpu.util,5m)>90 Haute Alerte e-mail admin
Ping pfSense échoue Pfsensee nodata(/Pfsensee/icmpping,5m)=1 Haute Alerte réseau
Disque C: > 90% utilisé Aura-AD last(/AuraAD/vfs.fs.pused[C:])>90 Avertissement Notification + ticket GLPI
Swap > 50% utilisé Zabbix server last(/Zabbix server/system.swap.pfree)<50 Info Journal + notification
