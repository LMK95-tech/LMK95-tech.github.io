01Analyse & Contexte
1.1 — Réseau LAN non protégé : risques critiques

Eco-Tech Solutions dispose d'un réseau LAN composé de 50 postes utilisateurs et d'un serveur Web interne Debian hébergeant le portail applicatif. Avant l'intervention, les flux Internet transitaient via un simple routeur FAI sans filtrage — le serveur Web n'était accessible que depuis le LAN.

 4 risques critiques identifiés
 IPs privées exposées
Trames LAN atteignant directement le WAN — topologie interne révélée
 Aucun filtrage entrant
Zéro règle bloquant les connexions entrantes — vecteur d'attaque maximal
 Serveur Web inaccessible
Absence de redirection — portail inaccessible depuis l'extérieur
 Aucune traçabilité
Sans pare-feu, aucun journal d'événements réseau disponible
1.2 — Solution retenue : passerelle pfSense 2.7.2

Installation d'un firewall pfSense 2.7.2 positionné en coupure entre le réseau WAN (FAI) et le LAN. pfSense assure trois fonctions critiques : NAT Outbound (masquerading), PAT/Port Forwarding (redirection 80/443 vers le serveur Web interne), et filtrage ACL (contrôle précis des flux autorisés).

 Bénéfices pfSense
IP publique unique 82.124.15.67 masque l'ensemble des 50 postes internes
Serveur Web 192.168.10.50 accessible depuis Internet via Port Forwarding
Logs centralisés : chaque connexion enregistrée pour audit de sécurité

03NAT Outbound — Masquerading
3.1 — Configuration NAT Outbound (Mode Hybride)

Le NAT Outbound (sortant) est configuré en mode Hybride dans Pare-feu → NAT → Sortant. Une règle spécifie que tout trafic provenant de 192.168.10.0/24 vers n'importe quel hôte externe est traduit (masqué) vers l'adresse WAN 82.124.15.67. Les 50 postes LAN sont ainsi invisibles depuis Internet.

pfSense ShellVérification table NAT active — pfctl
# Vérification des règles NAT actives depuis le shell pfSense
root@Eco-Tech-FW01:~# pfctl -s nat
# Résultat — NAT Outbound Masquerade
nat on em0 from 192.168.10.0/24 to any -> (em0)
# Résultat — PAT Port Forwarding HTTP/HTTPS
rdr on em0 proto tcp from any to 82.124.15.67 port 80 -> 192.168.10.50
rdr on em0 proto tcp from any to 82.124.15.67 port 443 -> 192.168.10.50
✓ NAT Outbound + PAT HTTP/HTTPS opérationnels
TypeInterfaceSource → DestinationTranslationDescription
NAT Outbound WAN 192.168.10.0/24 → Any 82.124.15.67 Sortie Internet LAN
PAT HTTP WAN Any → WAN:80 192.168.10.50:80 Serveur Web interne
PAT HTTPS WAN Any → WAN:443 192.168.10.50:443 Serveur Web interne (SSL)
04PAT / Port Forwarding
4.1 — Exposition contrôlée du serveur Web

Le Port Forwarding (DNAT) est configuré dans Pare-feu → NAT → Transfert de port. Deux règles redirigent les flux entrants sur l'IP publique 82.124.15.67:80 et :443 vers le serveur Web interne 192.168.10.50:80. pfSense réalise la translation de manière totalement transparente.

05Règles de filtrage LAN (ACL)
5.1 — Principe du moindre privilège réseau

Les règles de filtrage LAN définissent précisément les flux autorisés depuis le réseau interne. Seuls le DNS (port 53), le HTTP (port 80) et le HTTPS (port 443) sont explicitement autorisés. Tout autre trafic est bloqué par la règle implicite de refus — ce principe de moindre privilège réseau limite drastiquement la surface d'attaque.

ProtocoleSourcePort Dest.DescriptionAction
IPv4 TCP/UDP LAN net 53 (DNS) Autoriser DNS LAN ✓ Autoriser
IPv4 TCP LAN net 80 (HTTP) Autoriser HTTP LAN ✓ Autoriser
IPv4 TCP LAN net 443 (HTTPS) Autoriser HTTPS LAN ✓ Autoriser
IPv4 Any LAN net Any Tout bloquer (règle implicite) ✖ Bloquer
