01. Contexte & Problématique

L'établissement scolaire client d'Orange accueille 300 élèves et 50 personnels administratifs, répartis sur deux zones fonctionnelles distinctes. Avant l'intervention, l'ensemble du parc informatique fonctionnait en adressage IP entièrement statique, ce qui entraînait des dysfonctionnements critiques au quotidien.

Périmètre Client

  • 300 élèves (VLAN 3)
  • 50 personnels (VLAN 2)
  • Routeurs Cisco 2911
  • Switchs WS-C2950T-24
  • Simulateur Packet Tracer

Problèmes identifiés

  • Conflits d'adresses IP fréquents
  • Passerelles saisies incorrectement
  • ~15 min de config manuelle/poste
  • Aucune traçabilité centralisée
  • Charge administrative élevée
Scénario type d'incident
Nouveau poste branché IP 192.168.2.12 déjà utilisée Conflit — réseau inaccessible

02. Solution Retenue

La solution choisie est le déploiement d'un service DHCP embarqué directement sur les routeurs Cisco (Router0 et Router1), couplé à une encapsulation 802.1Q pour le routage inter-VLAN et au protocole RIPv2 pour la propagation des routes entre les deux sites.
DHCP Embarqué
Pools vlan2 & vlan3 sur IOS
802.1Q / dot1Q
Encapsulation sous-interfaces
RIPv2
Routage dynamique inter-sites

Gains attendus

Attribution automatique : adresse IP, masque, passerelle et serveur DNS via un seul pool nommé.
Élimination totale des conflits d'adresses grâce à la gestion centralisée des baux.
Réseau de transit 172.16.0.0/24 assurant la communication Router0 ↔ Router1.
Temps de configuration d'un nouveau poste réduit de 15 minutes à moins d'une minute.

05. Configuration Router0 — VLAN 2 (Admin)

Router0 héberge le pool DHCP vlan2. La sous-interface Gi0/1.2 est encapsulée en dot1Q 2 pour communiquer avec Switch0 sur le VLAN 2. RIPv2 est activé pour propager les routes.
Router0 — IOS Command Line Interface
! ── Interface Transit (Gi0/0) ────────────────── Router(config)#int g0/0 Router(config-if)#no shutdown Router(config-if)#ip address 172.16.0.253 255.255.255.0 ! ── Sous-interface VLAN 2 (dot1Q) ────────────── Router(config-if)#int g0/1 Router(config-if)#no shutdown Router(config-if)#int g0/1.2 Router(config-subif)#encapsulation dot1Q 2 Router(config-subif)#ip address 192.168.2.254 255.255.255.0 ! ── Pool DHCP vlan2 ──────────────────────────── Router(config-subif)#ip dhcp pool vlan2 Router(dhcp-config)#network 192.168.2.0 255.255.255.0 Router(dhcp-config)#dns-server 8.8.8.8 Router(dhcp-config)#default-router 192.168.2.254 ! ── Routage RIPv2 ────────────────────────────── Router(dhcp-config)#router rip Router(config-router)#version 2 Router(config-router)#network 192.168.2.0 Router(config-router)#network 192.168.3.0 Router(config-router)#network 172.16.0.0 Router(config-router)#end Router#write memory %SYS-5-CONFIG_I: Configured from console by console

06. Configuration Router1 — VLAN 3 (Élèves)

Router1 héberge le pool DHCP vlan3. La sous-interface Gi0/1.3 est encapsulée en dot1Q 3. Le même réseau de transit 172.16.0.0 est annoncé via RIPv2 pour assurer la connectivité inter-sites.

07. Configuration Switchs Catalyst

Les deux Switchs WS-C2950T-24 sont configurés pour créer les VLANs, définir le port uplink vers le routeur en mode trunk (Gi0/1) et affecter les ports d'accès (fa0/1-5) aux VLANs correspondants.
